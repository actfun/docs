Every token page on ACTFUN includes a built-in live voice room powered by **LiveKit**. The creator can broadcast camera and microphone to all viewers. Miners join and talk back by voice (audio-only — no viewer video). A live mining feed below the call shows real-time on-chain activity.

<Info>
  Only the wallet that created the token sees the **Go Live** button. Viewers see the stream controls when it is active.
</Info>

## How it works

ACTFUN uses **LiveKit** for all media, presence, and chat. Each token gets its own room (room name = launcher address, lowercased). The API server mints access tokens server-side with role-based permissions.

### Roles

| Role | Can publish | Can subscribe | Can chat |
| --- | --- | --- | --- |
| **Creator** | camera + microphone | yes | yes |
| **Miner (viewer)** | microphone only | yes | yes |

When a creator goes live:

1. The creator's browser captures camera and microphone.
2. LiveKit publishes the video + audio tracks to the room.
3. Miners join the same room, publish their microphone, and receive the creator's video + everyone's audio.
4. A **live mining feed** below the call renders `ActedFun` events in real time.
5. A **live chat** panel lets everyone send text messages via LiveKit data messages.
6. A **viewer count** updates in real time.

## Viewer experience

Miners who land on a token page while a stream is active see:

- The creator's video stream at the top of the token detail page.
- A **Join Voice** button to connect their microphone and talk.
- A **live mining feed** showing real-time `ActedFun` events (who mined, what they posted, when).
- A **live chat** panel for text messages.
- A pulsing "🔴 LIVE" badge on the token card.

If the creator leaves, the stream ends for all viewers.

## Starting a stream

<Steps>
  <Step title="Open your token's detail page">
    Navigate to your token on [actfun.fun](https://actfun.fun). The **Go Live** button appears in the top-right corner of the token detail page only for the wallet that created the token.
  </Step>
  <Step title="Sign the auth message">
    The app asks you to sign a short message proving you control the creator wallet. The server verifies the signature before minting the LiveKit token.
  </Step>
  <Step title="Click Go Live">
    Click **Go Live**. Your browser will ask for permission to access your camera and microphone. Grant the permissions to continue.
  </Step>
  <Step title="You're live">
    The stream starts immediately. The “🔴 LIVE�� badge appears for all viewers. Miners can join voice and chat.
  </Step>
  <Step title="End the stream">
    Click **End Stream** to stop broadcasting. The stream terminates for all viewers instantly.
  </Step>
</Steps>

## Authentication

LiveKit access tokens are minted server-side by the API server (`POST /api/live/:launcherAddress/token`). The request body includes:

- `identity` — the wallet address
- `role` — `"creator"` or `"viewer"`
- `signature` — a signed auth message (required for creator role)

The server verifies the signature recovers to the claimed identity, and for the creator role, additionally checks that the identity matches `launcher.creator()` on-chain. This prevents spoofed creator broadcasts.

## Technical details

| Detail | Value |
| --- | --- |
| Platform | LiveKit (livekit-client + livekit-server-sdk) |
| Room name | Launcher address (lowercased) |
| Creator tracks | camera + microphone |
| Miner tracks | microphone only (audio-only) |
| Chat | LiveKit data messages |
| Mining feed | `useLauncherEvents` hook (type = `"mine"`) |
| Auth | Wallet signature + on-chain creator verification |
| Persistence | Session-only (no recording or replay) |
