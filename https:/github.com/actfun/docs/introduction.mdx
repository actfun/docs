ACTFUN is a Pump.fun-style token launchpad on Arc Testnet. Instead of buying into a presale or hoping for an airdrop, you earn tokens by doing something simple: writing a funny post and paying a small USDC fee. The community mines 95% of the supply directly to their own wallets. When mining completes, the token automatically opens for live trading on its own built-in DEX - no external exchange, no admin action required.

## The Mine-to-Launch concept

Every token on ACTFUN follows the same four-stage lifecycle:

<Steps>
  <Step title="Create">
    Anyone can deploy a new token via the LaunchpadFactory. The creator sets the name, symbol, total supply, how many tokens each mine yields, a per-wallet cooldown, a daily cap, and a USDC fee per mine.
  </Step>
  <Step title="Mine">
    Community members mine the token by writing a funny post and paying the USDC fee. Each successful mine mints tokens directly to the miner's wallet. Cooldowns and daily caps, enforced entirely in the smart contract, keep things fair.
  </Step>
  <Step title="Graduate">
    When 95% of the supply has been mined, the contract automatically graduates. It mints the remaining 5% as LP reserve tokens, then opens a constant-product AMM (x\*y=k) seeded with those tokens plus all accumulated USDC fees.
  </Step>
  <Step title="Trade">
    Once graduated, anyone can buy or sell the token using the AMM built into the same contract. No bridging, no external DEX, no extra setup needed.
  </Step>
</Steps>

## Key benefits

<CardGroup cols={2}>
  <Card title="100% community mined" icon="pickaxe">
    Every token, aside from the 5% LP reserve minted at graduation, goes directly to the people who mine it. There is no team allocation, no insider distribution, and no presale.
  </Card>

  <Card title="Zero team allocation" icon="shield-check">
    The creator deploys the contract and sets the parameters, but receives no special token grant. The same mining rules apply to everyone.
  </Card>

  <Card title="Fully onchain" icon="link">
    All supply caps, cooldowns, daily limits, graduation triggers, and AMM logic live in the smart contracts on Arc Testnet. Nothing is controlled by an off-chain server or admin key.
  </Card>

  <Card title="No backend API" icon="server">
    The app communicates directly with Arc Testnet via RPC. There is no database, no centralized server, and no single point of failure.
  </Card>
</CardGroup>
